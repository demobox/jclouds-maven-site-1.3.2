69     /**
70      * The list hardware profiles command shows you the options including virtual cpu count, memory,

72      * differently. However, it is a good indicator of relative speed within a cloud. memory is

79     /**

82      * reason, you should consider matching image requirements like operating system family with
83      * TemplateBuilder as opposed to choosing an image explicitly. The getImages() command returns a
84      * map of images by id.

92     Set<? extends ComputeMetadata> listNodes();

94     /**
95      * The list locations command returns all the valid locations for nodes. A location has a scope,
96      * which is typically region or zone. A region is a general area, like eu-west, where a zone is
97      * similar to a datacenter. If a location has a parent, that implies it is within that location.
98      * For example a location can be a rack, whose parent is likely to be a zone.
99      */
100    Set<? extends Location> listAssignableLocations();

102    /**
103     * 
104     * The compute api treats nodes as a group based on the name you specify. Using this group, you
105     * can choose to operate one or many nodes as a logical unit without regard to the implementation
106     * details of the cloud.
107     * <p/>
108     * 
109     * The set that is returned will include credentials you can use to ssh into the nodes. The "key"
110     * part of the credentials is either a password or a private key. You have to inspect the value
111     * to determine this.
112     * 
113     * <pre>
114     * if (node.getCredentials().key.startsWith("-----BEGIN RSA PRIVATE KEY-----"))
115     *    // it is a private key, not a password.
116     * </pre>
117     * 
118     * <p/>
119     * Note. if all you want to do is execute a script at bootup, you should consider use of the
120     * runscript option.
121     * <p/>
122     * If resources such as security groups are needed, they will be reused or created for you.
123     * Inbound port 22 will always be opened up.
124     * 
125     * @param group
126     *           - common identifier to group nodes by, cannot contain hyphens
127     * @param count
128     *           - how many to fire up.
129     * @param template
130     *           - how to configure the nodes
131     * @return all of the nodes the api was able to launch in a running state.
132     * 
133     * @throws RunNodesException
134     *            when there's a problem applying options to nodes. Note that successful and failed
135     *            nodes are a part of this exception, so be sure to inspect this carefully.
136     */
137    Set<? extends NodeMetadata> createNodesInGroup(String group, int count, Template template) throws RunNodesException;
138
